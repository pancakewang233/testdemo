# 项目配置指南

## 环境要求

- Node.js >= 14.0.0
- npm >= 6.0.0

## 安装步骤

### 1. 安装 Node.js

如果您的系统未安装 Node.js，请按以下步骤操作：

1. **下载 Node.js**
   - 访问 [Node.js 官网](https://nodejs.org/)
   - 下载 LTS（长期支持）版本
   - 推荐版本：v18.x 或 v20.x

2. **安装 Node.js**
   - 运行下载的安装程序
   - 按照向导完成安装（默认会添加到 PATH 环境变量）
   - 安装完成后会自动包含 npm

3. **验证安装**
   打开 PowerShell 或命令提示符，运行：
   ```bash
   node --version
   npm --version
   ```

### 2. 安装项目依赖

在项目根目录下运行：

```bash
npm install
```

这将安装以下依赖：
- Vue 2.6.14
- Element UI 2.15.14
- Webpack 及相关构建工具

### 3. 运行项目

开发模式（带热重载）：
```bash
npm run dev
```

项目将在 `http://localhost:8080` 启动，浏览器会自动打开。

### 4. 构建生产版本

```bash
npm run build
```

构建产物将输出到 `dist` 目录。

## 项目配置说明

### Webpack 配置
- 开发服务器端口：8080
- 支持热模块替换（HMR）
- 自动打开浏览器

### Element UI 配置
- 已在 `src/main.js` 中全局引入
- 样式文件已自动加载
- 所有组件可直接使用

### Babel 配置
- 使用 `@babel/preset-env` 进行 ES6+ 转译
- 支持现代 JavaScript 语法

## 常见问题

### 问题1：npm 命令无法识别
**解决方案**：
- 确认 Node.js 已正确安装
- 重启终端或 IDE
- 检查环境变量 PATH 是否包含 Node.js 安装路径

### 问题2：端口 8080 被占用
**解决方案**：
- 修改 `webpack.config.js` 中的 `devServer.port` 为其他端口（如 8081）
- 或关闭占用 8080 端口的程序

### 问题3：Element UI 样式未加载
**解决方案**：
- 确认 `src/main.js` 中已引入样式文件：`import 'element-ui/lib/theme-chalk/index.css'`
- 重新运行 `npm install` 确保依赖完整
